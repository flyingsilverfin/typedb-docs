= `label` statement
:page-aliases: {page-version}@typeql-reference::statements/type.adoc
:test-typeql: linear

The constraint `$t label <LABEL>` matches all types `$t` that have the label (name) `<LABEL>`.

== Defining type labels

The `label` keyword is only used when matching on a type label. The type label is created automatically when the type is defined.

[,typeql]
----
#!test[schema]
define entity user;
----

== Matching type labels

[,typeql]
----
#!test[schema]
match $t label user;
----

== The label() function

The `label()` function returns the label of a type or role as a string in an expression context.

[,typeql]
----
match
  $_ isa $t;
  let $label = label($t);
----

NOTE: For roles, the label includes the relation namespace (e.g. `"rel:rolename"`).

See xref:{page-version}@typeql-reference::expressions/operators.adoc#_concept_functions[] for more details.
